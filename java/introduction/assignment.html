Chapter 5

Assignment

Assignment always copies what the variable holds. That one rule — applied to primitives and references — explains everything about how values move through a Java program.

We have seen assignment twice already: once for primitive variables in §2, and once for reference variables in §4.2. This chapter names the pattern explicitly, because it will matter in §6 when method calls are introduced.

Primitive assignment — copies the value

When you assign one primitive variable to another, the value in the slot is copied. You end up with two independent slots, each holding the same value.

int a = 10;
int b = a;   // b gets its own slot; the value 10 is copied into it

b = 99;      // modifies b's slot only

System.out.println(a); // 10 — unchanged
System.out.println(b); // 99
Memory model

Before: one slot named a containing 10. After int b = a: two slots, a and b, each independently containing 10. They share no connection — changing one has no effect on the other.

Reference assignment — copies the address

When you assign one reference variable to another, the address in the slot is copied. You end up with two variables holding the same address — two names for the same object.

Rectangle r1 = new Rectangle();
r1.width = 10;

Rectangle r2 = r1;  // r2 gets its own slot; the address in r1 is copied into it
Memory model

Before: r1 holds address 0x4a2f. After Rectangle r2 = r1: two slots, r1 and r2, both containing 0x4a2f. One object on the heap. Two variables pointing at it.

Consequence: modifications are visible through both variables

Because both variables point to the same object, modifying the object through one variable is immediately visible through the other.

r2.width = 99;      // dereferences r2, modifies the object

System.out.println(r1.width); // 99 — r1 sees the change too
System.out.println(r2.width); // 99

This is not a quirk or a bug — it follows directly from the fact that assignment copied the address. There is one object. Both variables hold its address. Whoever writes to it, the other sees the result.

Contrast with primitives

With primitives, assignment copies the value — two independent slots, no connection. With references, assignment copies the address — one object, two names. The behavior after assignment is completely different depending on which kind of variable you're working with.

The rule

Assignment always copies what the variable holds:

  • Primitive variable → copies the value. Two independent slots.
  • Reference variable → copies the address. Two names for one object.

There is no third case. Every assignment in Java — whether it appears as a declaration, a standalone statement, or inside a more complex expression — follows one of these two forms.

Looking ahead to §6

When a method is called, the argument is assigned to the parameter. When the method returns, the return value is assigned to the receiving variable. These are ordinary assignments — the same two rules apply. Knowing this now makes method calls in §6 straightforward rather than surprising.
