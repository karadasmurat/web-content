Chapter 3

Classes

A class is a blueprint — a description of what fields every object of that kind will have. The class itself is not an object; it describes objects.

So far, every variable we've seen holds a single primitive value: a number, a boolean. Most interesting data is made of several things grouped together. A rectangle has a width and a height. A point has an x and a y. A class lets you define that grouping.

A class is a description of a kind of object: it names the fields that every object of that kind will have. Think of it as a blueprint. The blueprint is not a building — it describes buildings.

Defining a class

Our first class uses only primitive fields — deliberately:

class Rectangle {
    int width;
    int height;
}

This declares that every Rectangle will have two slots: width (an int) and height (an int). The class definition itself occupies no heap memory. It is a template.

The field declarations inside the class look exactly like the variable declarations from §1 — because they are. They're just variables that belong to an object rather than to a method call. Once objects exist (§4.1), each one will have its own independent width and height slots.

Multiple instances, independent slots

The power of a class is that you can create many objects from the same blueprint, each with its own field values:

// Two separate Rectangle objects — each has its own width and height
// (creating objects requires 'new', covered in §4.1)
Rectangle r1 = new Rectangle();
Rectangle r2 = new Rectangle();

r1.width = 10;
r2.width = 50;  // r1.width is still 10 — r1 and r2 are independent objects
Deliberate constraint

Both fields are primitives. We are specifically not adding any field whose type is a class — no field that would hold an address. That constraint is intentional. We will lift it in §4.3, once reference variables are understood. The moment reference fields appear will be clearly marked.

Memory model

The class definition lives in the program's type information — it takes up no heap memory. When you create an object with new (§4.1), Java allocates a chunk of heap memory whose layout is determined by the class: one slot for each field, sized according to field types.
