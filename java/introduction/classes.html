Chapter 3

Classes

A class is a blueprint. The class itself is not an object; it describes objects.

So far, every variable we've seen holds a single primitive value: a number, a boolean. Most interesting data is made of several things grouped together — the way a real-world kind of thing is described by its attributes. A rectangle has a width and a height. A point has an x and a y. A person has a name and an age. A class lets you define that grouping: the set of attributes that together describe one kind of thing.

A class is a description of a kind of object: it names the fields that every object of that kind will have. Think of it as a blueprint — the way an architect draws up a detailed technical drawing describing a house: its dimensions, the number of rooms, where the doors go. The blueprint itself is not a house; you can't live in it. But it's precise enough that a builder can follow it, and many separate houses can be built from that same one blueprint. Each has the same rooms in the same layout — but one is painted blue, another has hardwood floors, another has three bedrooms filled and one empty. Same structure, different values in each house. A class works the same way for objects.

Defining a class

Our first class uses only primitive fields:

class Rectangle {
    int width;
    int height;
}

This declares that every Rectangle will have two slots: width (an int) and height (an int). The class definition itself is a template — no Rectangle object exists yet, just the description of what one will look like.

The field declarations inside the class look exactly like the variable declarations from §1 — because they are. They're just variables that belong to an object rather than to a method call. Once objects exist (§4.1), each one will have its own independent width and height slots.
