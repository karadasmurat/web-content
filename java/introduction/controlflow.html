Chapter 10

Control Flow

Every program so far has run top to bottom, once, every line. Control flow is how you make a boolean (§9) decide whether a line runs at all, or how many times it repeats.

if / else — conditional execution

An if runs a block only when its condition is true. The condition is any expression that produces a boolean — exactly the comparison and logical operators from §9:

int width = 10;

if (width > 5) {
    System.out.println("wide");
}

else runs instead, when the condition is false. else if chains additional conditions, checked in order, top to bottom — the first one that's true runs, and the rest are skipped:

int width = 10;

if (width > 20) {
    System.out.println("wide");
} else if (width > 5) {
    System.out.println("medium");
} else {
    System.out.println("narrow");
}
// medium
Only a boolean

Java requires the condition to be a genuine boolean — not an int standing in for one, the way some languages allow. if (width) is a compile error unless width is itself declared boolean. If you mean "not zero," you write if (width != 0).

while — repeat while a condition holds

A while loop re-checks its condition before every run of its block, and stops the moment that condition is false:

int i = 0;
while (i < 5) {
    System.out.println(i);
    i++;
}
// 0 1 2 3 4

Three moving parts make this work, and all three are things you already know: i starts at a known value, the condition (§9's <) is checked before each iteration, and i++ (§9) changes the value the condition depends on. Leave that last part out, and the condition never becomes false — the loop runs forever.

for — the loop with its bookkeeping up front

A for loop is a while loop with its three moving parts gathered into one line: initialize, then condition, then update.

for (int i = 0; i < 5; i++) {
    System.out.println(i);
}
// 0 1 2 3 4

Read the parentheses as three statements separated by ;: int i = 0 runs once, before anything else. i < 5 is checked before every iteration, exactly like while. i++ runs after every iteration, before the condition is checked again. This is precisely the previous example, rearranged — nothing new is executing, it's just harder to forget the update when it lives next to the condition.

Now the array chapter's opening example can finally be written the way it was always heading:

int[] temperatures = {68, 71, 69, 73, 75, 70, 66};

int total = 0;
for (int i = 0; i < temperatures.length; i++) {
    total += temperatures[i];
}
double average = total / (double) temperatures.length;
System.out.println(average); // 70.28571428571429
Same idea as day1 + day2 + ... ← §7

§7 opened by hand-writing day1 + day2 + ... + day7 — one variable per value, and a sum that has to name every one of them. temperatures[i], with i counting 0 through temperatures.length - 1, reaches every element the same way that hand-written sum did, without a fixed number of terms baked into the code. Ask for thirty temperatures instead of seven, and this loop doesn't change at all.

for-each — when you don't need the index

Most of the time, a loop over an array wants every element, in order, and never actually uses i for anything except temperatures[i]. The for-each loop drops the index entirely:

int total = 0;
for (int temp : temperatures) {
    total += temp;
}

Read for (int temp : temperatures) as: "for each int in temperatures, call it temp." Every iteration, temp holds the next element's value — copied out, exactly like any other primitive assignment. There's no i to initialize, check, or increment, because there's no index at all.

When you still need the classic for

For-each can't skip elements, go backwards, or change which element it's looking at — it always visits every element, front to back, once. It also can't tell you the index of the element it's holding. Need any of that — or need to modify the array itself through the loop variable — and the classic for (int i = 0; ...) is still the right tool.

Scope of this chapter

switch, do-while, and the break/continue statements that fine-tune a loop mid-run all build on exactly the ideas here — a boolean condition, and a block that runs some number of times — and are left for later.
