Chapter 2

Primitives

The simplest kind of variable: the slot directly contains the value. Just the data itself.

Java has eight primitive types. A variable's data type determines the values it may contain.

When a variable is of a primitive type, the slot is the value. There is no indirection — think of it as a labeled box with the actual data sitting directly inside, not a box holding directions to where the data lives. Open the box, and the value is right there.

The word value is going to show up constantly from here on, so it's worth being precise about what it means the moment it's said of a primitive variable: the data itself, occupying the slot, with nothing standing between the variable and what it holds.

int     width  = 10;    // slot named 'width'  contains 10
double  price  = 3.99;  // slot named 'price'  contains 3.99
boolean active = true;  // slot named 'active' contains true

All eight primitive types:

Type Description Size Value range
byte Whole number 8-bit -128 to 127
short Whole number 16-bit -32,768 to 32,767
int Whole number 32-bit -2,147,483,648 to 2,147,483,647
long Whole number, for values bigger than int can hold 64-bit -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807
float Decimal number, lower precision 32-bit ~ ±3.4 × 1038, about 6–7 significant decimal digits
double Decimal number 64-bit ~ ±1.8 × 10308, about 15–16 significant decimal digits
boolean Exactly one of two values 1 bit's worth of information true or false — nothing else
char A single Unicode character 16-bit 0 to 65,535 (one UTF-16 code unit — §10 covers what that means)

Every one of these types also has a default value — what a field of that type starts out holding, before any constructor runs. §4.4 covers default values in full, once fields and constructors are actually on the table.

Literals

Every value written directly into source code — 10, 3.99, true, 'A' — is a literal: a fixed value, spelled out in the code itself, rather than computed or read from a variable. A literal's own syntax is what tells the compiler which type it is, without needing a declared variable anywhere nearby:

Kind Example Notes
Decimal integer 42, -7 Base 10 — the default form, and an int unless suffixed otherwise
Hexadecimal integer 0x2A Prefixed 0x, base 16 — 0x2A is the same value as 42
Binary integer 0b101010 Prefixed 0b, base 2 — also 42
Long 10000000000L Trailing L forces 64-bit, overriding the 32-bit int default
Double 3.14 The default for any decimal literal — a trailing D is legal but redundant
Float 3.14f Trailing f forces 32-bit, overriding the double default
Boolean true, false Exactly these two words — nothing else is a boolean literal
Character 'A', '\n', '\u0041' Single quotes; escape sequences like \n and Unicode escapes like \u0041 are both legal — '\u0041' is the same character as 'A'

Each of those literals is typically assigned straight into a variable of the matching type:

int   decimal = 42;
int   hex     = 0x2A;
int   binary  = 0b101010;
long  big     = 10000000000L;
double price  = 3.14;
float ratio   = 3.14f;
boolean done  = true;
char  letter  = 'A';
char  newline = '\n';
char  alsoA   = '\u0041';

Notice each literal's own form already matches the declared type on its left — 0x2A and 0b101010 are both just 42 written differently, and a variable declared int accepts either one without any conversion, because the value they produce is identical. Get the pairing wrong — say, assigning 3.14 to an int without a suffix telling the compiler otherwise — and it's a compile error, not a rounding surprise at runtime.

Underscores for readability

A numeric literal can use underscores to group digits — 1_000_000 is exactly the same int as 1000000. The compiler strips the underscores entirely; they exist purely to make a long literal easier for a person to read at a glance.

What "N-bit" actually means

A bit is a single binary digit — 0 or 1, the smallest unit of storage there is. A slot built from N bits can hold exactly 2N distinct patterns, and not one more — that's the entire rule, for every type above. 8 bits give 28 = 256 distinct patterns, which is exactly byte's range: 256 values, split to run from -128 to 127. 16 bits give 216 = 65,536 patterns — short's range, and, for the same reason, char's: a char is a 16-bit slot, so it can represent exactly 65,536 distinct values, full stop, regardless of what those values are being used to mean.

That ceiling matters more than it looks like it should — enough to be worth its own chapter (§10) once char is asked to represent something bigger than 65,536 possibilities actually turns out to be.

Basic arithmetic and operations

int width  = 10;
int height = 20;
int area   = width * height;      // 200
int perim  = 2 * (width + height); // 60

double half = perim / 2.0;         // 30.0
boolean big = area > 100;          // true

Two things are worth noticing here, not just the arithmetic itself. First, perim / 2.0 divides an int by a double literal — the result is a double, so half comes out 30.0, not 30; dividing by a plain 2 instead would have triggered integer division and truncated any fractional part. Second, area > 100 is itself a value — a boolean one — computed once and stored in big, exactly like any other primitive assignment above it.

Why this matters now

The word "value" appears constantly from this point on. When we say a primitive variable holds a value, we mean the data lives directly in the slot — no indirection. That's all. Hold this contrast in mind; it becomes important when reference variables arrive in §4.2.
