Chapter 2

Primitives

The simplest kind of variable: the slot directly contains the value. No indirection, no heap, no addresses. Just the data itself.

Java has eight primitive types. Three are enough to ground the concept: int, double, and boolean.

When a variable is of a primitive type, the slot is the value. There is no indirection.

int     width  = 10;    // slot named 'width'  contains 10
double  price  = 3.99;  // slot named 'price'  contains 3.99
boolean active = true;  // slot named 'active' contains true

The three primitives used in this tutorial:

  • int — a whole number: 0, 10, -42, 1000000.
  • double — a number with a decimal point: 3.14, -0.5, 2.0.
  • boolean — exactly one of two values: true or false.

Copying a primitive copies the value

Assigning one primitive variable to another copies the contents of the slot — you end up with two independent slots, each holding the same value.

int a = 10;
int b = a;    // b gets its own slot; contents copied from a

b = 99;       // changes b's slot only

System.out.println(a); // 10 — a is unaffected
System.out.println(b); // 99
Memory model

After int b = a: two slots, two labels, both containing 10. They share no connection. Changing one has no effect on the other.

Basic arithmetic and operations

int width  = 10;
int height = 20;
int area   = width * height;      // 200
int perim  = 2 * (width + height); // 60

double half = perim / 2.0;         // 30.0
boolean big = area > 100;          // true
Why this matters now

The word "value" appears constantly from this point on. When we say a primitive variable holds a value, we mean the data lives directly in the slot — no indirection. That's all. Hold this contrast in mind; it becomes important when reference variables arrive in §4.2.
