Chapter 1

Variables

A variable is a named slot in memory, and it holds exactly one of two things: a value, stored directly, or an address, pointing elsewhere. That distinction drives everything that follows.

Your program runs in a computer that has memory: a vast array of numbered slots, each holding a small piece of data. A variable is your way of naming one of those slots so you can refer to it by name rather than by its raw address.

You declare a variable by giving it a type and a name:

int count;
boolean active;

This reserves named slots in memory. The type tells Java two things: how large each slot should be, and what kind of data it will hold.

Every variable in Java holds exactly one of two things:

  • A value — the data itself, stored directly in the slot. This is what primitives do (§2).
  • An address — the location of some data stored elsewhere in memory. This is what reference variables do (§4.2).

You can also assign an initial value at the point of declaration:

int count   = 0;
boolean active = true;
Memory model

Think of a variable as a labeled slot. The label is the variable name. The slot has a fixed size determined by the type. For now, the content of the slot is either a value (§2) or an address (§4.2).

Scope of this chapter

We're not yet talking about what types are available or what operations exist. Just the fundamental idea: a variable is a named slot, and that slot holds either a value or an address. That distinction is the thread running through the entire tutorial.
